## 컨테이너 오케스트레이션 선택

### Option A — ECS Fargate (실사용 권장)

- AWS 네이티브, 설정 간단, 운영 오버헤드 낮음
- CodeDeploy 연동으로 자동 배포 용이
- 발표 포인트: 빠른 배포, AWS 서비스 완전 통합

### Option B — EKS (발표용 시연 가능)

- Kubernetes 기반, 멀티클라우드 이식성
- `deployment.yaml` / `service.yaml` / `ingress.yaml` 구성
- AWS Load Balancer Controller + IRSA 활용
- 발표 포인트: K8s 생태계 경험, 확장성 어필

| 항목 | ECS Fargate | EKS |
| --- | --- | --- |
| 설정 난이도 | 낮음 | 높음 |
| 운영 오버헤드 | 낮음 | 높음 |
| K8s 이식성 | ❌ | ✅ |
| 클러스터 비용 | 없음 | $0.10/시간 추가 |
| 발표 임팩트 | 보통 | 높음 |

> 실제 운영은 ECS, 발표 시연은 EKS로 병행 고려 가능
>

## 작업 순서 (예상)

### 전체 인프라

- VPC / 서브넷 / 보안 그룹 구성
- RDS 생성 (MySQL 8.0, 서비스별 4개 DB)
- DMS → MySQL EC2에서 RDS로 데이터 이전 (Full Load + CDC)
- ECS 클러스터 / Task Definition / Service 구성 (또는 EKS)
- ALB (internal) 구성 + ECS Target Group 연결
- ECR 리포지토리 생성 (서비스별)

### 배포 자동화

- Amplify (프론트) → GitHub 연동, API_URL / AZURE_MAPS_KEY 환경변수 설정
- CodePipeline (백엔드) → CodeBuild (docker build → ECR push) → CodeDeploy (ECS 재배포)
- 경로 필터 설정 (frontend/** / backend/** 분리 트리거)

### 보안

- IAM 사용자 → 팀원별 계정 생성 후 Root 계정 사용 중단
- Secrets Manager → DB 비밀번호, JWT 시크릿, Azure API 키 등록
- Cognito → User Pool 생성, API Gateway JWT Authorizer 연결
- WAF → API Gateway에 연결, SQL Injection / XSS / Rate Limiting 규칙 설정

### 로그

- CloudWatch → ECS 서비스별 로그 그룹 설정, 알람 구성
- S3 → CloudWatch 로그 Export 버킷 생성
- Athena → S3 로그 쿼리 테이블 구성
