## 안건 4. Terraform 협업 방식 개편 및 비용 관리 (논의)

**비용 최적화 + 협업 효율을 고려한 제안**

### 배경
인프라 가동 시간을 따로 정하지 않고 팀 계정에 24시간 켜두면 Aurora, ALB 등 일부 서비스가 사용 여부와 무관하게 지속 과금되어 크레딧이 예상보다 빠르게 소진될 수 있습니다.

### 제안
> Terraform 전체 구성 파일을 GitHub 저장소에 통합 관리하여,
> 팀원 각자가 **본인 AWS 계정**에서 자유롭게 테스트할 수 있는 환경을 만들자.

- 팀 계정은 **공식 배포 및 검수 용도**로만 사용, 평소 테스트는 개인 계정에서 진행
- `terraform.tfvars`의 계정 정보만 교체하면 동일한 인프라를 개인 계정에서 재현 가능
- 팀 크레딧 절약 + 각자 마음껏 테스트 가능

### 브랜치 전략 (제안)
- 기능별 브랜치 분리 후 작업 → PR 방식으로 통합
- 파일 일일이 주고받는 방식 지양

### Git GUI 툴 도입 (제안)
- **Sourcetree** 등 GUI 툴 사용으로 브랜치 가시성 확보
- 지난 기수 1팀도 동일 방식으로 협업
- Git 기본 사용법 간단히 공유 예정

## 안건 7. 협업 툴 적극 활용 요청

초반에 팀 일정 관리를 위해 구글 캘린더와 트렐로를 도입했지만 현재 활용도가 낮은 상태입니다.

프로젝트 마감이 다가올수록 일정 관리와 작업 현황 파악이 중요해지는 만큼 아래와 같이 적극적인 활용을 제안합니다.

| 툴 | 활용 방법 |
|----|----------|
| **트렐로** | 할 일 / 진행 중 / 완료 카드 업데이트 — 본인 작업 상태 실시간 반영 |
| **구글 캘린더** | 회의 일정, 마감 기한, 배포 일정 등록 |

> 작업 현황을 투명하게 공유해야 서로 의존하는 작업의 선후 관계를 파악하고 일정 지연을 예방할 수 있습니다.
